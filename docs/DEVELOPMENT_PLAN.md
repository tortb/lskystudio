# Lsky Studio 开发计划

## 项目定位

**"在保持现有 Node.js 业务逻辑基本不变的前提下，对项目进行现代化 UI 重构，并使用 Tauri 封装为跨平台桌面应用。"**

---

## 核心原则

1. **保留现有 Node.js 上传逻辑** - 不修改上传核心算法
2. **保留所有接口和业务流程** - 除非存在明显 Bug
3. **不允许为了美化 UI 而修改上传逻辑**
4. **Rust 仅负责 Tauri 外壳** - 启动 Node 子进程、系统托盘、窗口管理
5. **React 仅负责界面展示和用户交互**
6. **所有上传、配置、历史等业务继续由 Node.js 负责**
7. **新增代码必须保持模块化，不影响已有功能**

---

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Node.js 子进程通信问题 | 高 | 早期验证 IPC 通信 |
| Tauri 打包兼容性 | 中 | 多平台测试 |
| UI 与现有逻辑冲突 | 中 | 严格遵循接口约定 |
| 性能问题 | 低 | 优化渲染和数据加载 |
