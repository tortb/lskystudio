# 产品需求文档 (PRD)

## 1. 产品概述

**产品名称：** Lsky Studio

**产品定位：** 专业级兰空图床桌面客户端，支持批量上传、相册管理、历史记录等功能。

**目标用户：** 
- 内容创作者
- 开发者
- 设计师
- 需要频繁上传图片的用户

**产品价值：**
- 提供比 Web 端更高效的批量上传体验
- 支持断点续传和并发控制
- 本地化配置和历史管理
- 跨平台支持（Windows + Linux）

---

## 2. 核心功能

### 2.1 图片上传

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 批量选择文件 | P0 | 支持多选、拖拽、文件夹选择 |
| 并发上传 | P0 | 可配置并发数（1-20） |
| 进度显示 | P0 | 实时显示每个文件的上传进度 |
| 暂停/继续 | P0 | 支持暂停和继续上传 |
| 取消上传 | P0 | 支持取消正在进行的上传 |
| 断点续传 | P1 | 网络中断后可继续上传 |
| 文件压缩 | P1 | 上传前自动压缩图片 |
| 剪贴板上传 | P2 | 支持从剪贴板粘贴图片上传 |

### 2.2 相册管理

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 查看相册列表 | P0 | 获取远程相册列表 |
| 创建相册 | P1 | 创建新相册 |
| 上传到指定相册 | P0 | 选择相册后上传 |
| 相册缓存 | P1 | 本地缓存相册列表 |

### 2.3 历史记录

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 上传历史 | P0 | 记录所有上传历史 |
| 搜索历史 | P1 | 按文件名、日期搜索 |
| 复制链接 | P0 | 一键复制图片链接 |
| 导出报告 | P2 | 导出 Excel/CSV 报告 |
| 删除记录 | P1 | 删除本地历史记录 |

### 2.4 配置管理

| 功能 | 优先级 | 说明 |
|------|--------|------|
| API 配置 | P0 | 配置 API URL 和 Token |
| 存储策略 | P0 | 选择存储策略 |
| 代理设置 | P1 | 配置 HTTP 代理 |
| 重试策略 | P1 | 配置重试次数和间隔 |
| 主题切换 | P2 | 浅色/深色主题 |
| 语言设置 | P2 | 中文/英文 |

---

## 3. 非功能需求

### 3.1 性能要求

- 启动时间 < 2 秒
- 上传速度接近网络带宽上限
- 内存占用 < 200MB
- CPU 占用 < 10%（空闲时）

### 3.2 可靠性要求

- 上传成功率 > 99%
- 支持网络中断恢复
- 支持应用崩溃恢复
- 数据持久化（配置、历史）

### 3.3 安全要求

- Token 加密存储
- HTTPS 通信
- 本地数据加密（可选）

### 3.4 兼容性要求

- Windows 10/11
- Ubuntu 20.04+
- macOS 12+（可选）

## 4. 用户界面要求

### 4.1 设计风格

- 参考：VSCode、GitHub Desktop、Linear、Notion
- 风格：极简、现代、圆角、留白
- 布局：响应式，支持侧边栏折叠

### 4.2 主题支持

- 浅色主题（默认）
- 深色主题
- 跟随系统

### 4.3 页面结构

```
│  侧边栏 │           内容区域                    │
│  - 仪表盘│                                       │
│  - 上传  │                                       │
│  - 历史  │                                       │
│  - 相册  │                                       │
│  - 设置  │                                       │
│         │                                       │

```

---

## 5. 技术约束

- 使用 Tauri 2 构建桌面应用
- 使用 React + TypeScript 构建前端
- 使用 Node.js 处理上传逻辑
- 使用 SQLite 存储本地数据
- 使用 pnpm workspace 管理 monorepo
